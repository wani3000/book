# 회원 생애주기 UX·기능 감사

감사일: 2026-07-21

## 범위와 목표

비회원이 Google로 가입하고, 프로필과 구매내역을 관리하며, 환불·로그아웃·탈퇴·재가입까지 실수 없이 완료할 수 있는지 확인했다.

## 단계별 상태

1. 로그인 진입 — 양호
   - 전자책 구매내역과 PDF 관리라는 로그인 목적이 분명하다.
   - 기존 문구는 로그인만으로 약관 전체에 동의한 것으로 읽힐 위험이 있었다.
   - 수정 후 신규 계정에는 Google 인증 다음 필수 동의를 확인한다고 명시했다.
2. 신규 회원가입 동의 — 개선 완료
   - 이용약관과 개인정보 동의를 필수로 분리했다.
   - 마케팅 동의는 선택으로 분리하고 미동의 시 불이익이 없음을 표시했다.
   - 동의 일시와 정책 버전을 서버 DB에 기록한다.
3. 로그인·로그아웃 — 개선 완료
   - 활성 기존 회원은 반복 가입 절차 없이 로그인한다.
   - 여러 로그인 컴포넌트의 상태를 즉시 동기화한다.
   - 로그아웃 응답 실패 시 로그인 화면으로 잘못 전환하지 않고 오류를 표시한다.
4. 프로필 관리 — 양호
   - 표시 이름과 마케팅 수신 여부를 변경할 수 있다.
   - Google 이메일은 읽기 전용임을 안내한다.
5. 구매내역·PDF — 개선 완료
   - 주문번호, 결제수단, 결제금액, 구매일, 최초 열람, 주문·환불 상태를 표시한다.
   - 결제 완료 주문만 PDF 권한을 받고 환불 완료 시 권한을 회수한다.
6. 환불 — 양호
   - 환불 신청 사유, 조건 확인, 4단계 처리 상태가 연결돼 있다.
   - 처리 중 환불이 있으면 탈퇴를 차단해 결과 확인 권리를 보호한다.
7. 회원 탈퇴 — 개선 완료
   - 단순 확인창을 제거하고 결과 안내, `회원 탈퇴` 직접 입력, 확인 체크를 요구한다.
   - 탈퇴 즉시 세션과 PDF 접근을 중단하고 법정 거래 기록만 보관한다.
8. 재가입 — 개선 완료
   - 탈퇴 계정은 로그인만으로 자동 복구하지 않는다.
   - 기존 구매내역과 유효 권한 복원 사실을 알리고 필수 동의를 받은 뒤 재활성화한다.
9. 공개 정책·도움말 — 개선 완료
   - `/account-guide`, 이용약관, 개인정보처리방침의 탈퇴·재가입·기록 보관 설명을 동일하게 맞췄다.

## 접근성 확인

- 동의와 탈퇴 화면은 `role="dialog"`, `aria-modal`, 연결된 제목을 사용한다.
- 필수 체크박스와 직접 입력 필드에 화면 라벨이 있다.
- 오류는 `role="alert"`, 진행 결과는 기존 상태 영역으로 전달한다.
- 키보드 포커스 트랩, 화면낭독기 실제 발화, 고배율 리플로우는 스크린샷만으로 확정할 수 없어 공개 배포 뒤 실기기 점검이 필요하다.
