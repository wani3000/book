# My Page Journey QA 트리거 테스트

## Should trigger

1. 로그인 후 마이페이지에서 주문 내역을 누르면 아래에 붙어서 보여. 실제 페이지 이동인지 QA해줘.
2. `/mypage#profile` 방식이 맞는지 뒤로가기까지 전부 검수해.
3. 내 서재, 결제 내역, 회원정보 메뉴가 모바일에서 제대로 분리되는지 확인해줘.
4. 구매자는 PDF를 보고 환불 신청도 해야 해. 마이페이지 사용자 여정 전체 QA 부탁해.
5. 프로필을 눌렀는데 주문 카드가 같이 남아 있어. 라우팅 구조를 검사해.
6. 구글 로그인 후 원래 페이지로 복귀하고 마이페이지 하위 URL을 새로고침해도 되는지 테스트해.
7. 환불 완료된 계정에서 PDF 링크가 사라지는지 로그인 계정으로 QA해줘.
8. 관리자와 일반 회원의 마이페이지 메뉴가 다르게 보이는지 전수 검사해.
9. 결제 완료 페이지에서 구매 내역으로 가는 링크까지 포함해서 마이페이지 QA해.
10. 출시 전에 계정·주문·프로필 페이지를 데스크톱과 390 모바일로 검수해줘.

## Should not trigger

1. 메인 랜딩페이지의 카드 색상과 여백만 평가해줘.
2. 마이페이지 주문 API에 새 필드를 구현해줘. 검수는 필요 없어.
3. PDF 세 권의 오탈자와 표지 품질을 교정해줘.
4. 구글 OAuth 클라이언트 만드는 방법만 알려줘.
5. 네이버페이와 카카오페이 수수료를 비교해줘.
6. 회원가입 약관 문구를 법률 관점에서 검토해줘.
7. 관리자 환불 승인 API의 단위 테스트 하나를 작성해줘.
8. 전자책 상세페이지를 클래스101 스타일로 다시 디자인해줘.
9. Supabase가 무엇인지 초보자에게 설명해줘.
10. 현재 변경 사항을 커밋하고 GitHub에 푸시해줘.
