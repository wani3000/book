# 카카오 로그인 구현·운영 준비

최종 검토일: 2026-07-21

## 결론

카카오 로그인 코드와 안전한 계정 연결 구조는 구현되었다. 운영 키가 모두 설정되기 전에는 로그인 화면의 카카오 버튼이 자동으로 숨겨진다. 실제 활성화에는 카카오디벨로퍼스 앱 설정과 운영 환경값 등록, D1 마이그레이션 적용이 필요하다.

## 구현 완료 항목

1. `auth_identities` 테이블을 추가했다.
   - `provider`: `google` 또는 `kakao`
   - `provider_subject`: 로그인 제공자가 발급한 고유 사용자 ID
   - `member_id`: 다니엘의 노트 내부 회원 ID
   - `(provider, provider_subject)`를 고유값으로 둔다.
2. 기존 Google 회원은 `drizzle/0007_icy_silver_surfer.sql`에서 기존 회원 ID를 Google `provider_subject`로 자동 이관한다.
3. 카카오 REST API의 Authorization Code 방식으로 로그인한다.
   - 요청마다 `state`를 생성하고 콜백에서 일치 여부를 검증한다.
   - OpenID Connect를 활성화하면 `nonce`와 ID 토큰 서명·발급자·대상·만료를 검증한다.
   - 서버에서 인가 코드를 토큰으로 교환하며 클라이언트 시크릿은 서버 환경변수로만 보관한다.
4. 신규 카카오 사용자는 서비스 약관과 개인정보 수집·이용 동의를 거쳐 내부 회원을 만든다.
5. 로그인된 프로필 화면에서 Google·카카오 계정을 연결하거나, 다른 로그인 수단이 남아 있을 때 연결을 해제할 수 있다. 이메일이 같다는 이유만으로 자동 병합하지 않는다.
6. 탈퇴 시 서비스 회원을 비활성화하기 전에 카카오 플랫폼 연결 해제를 수행한다.

## 카카오디벨로퍼스 준비 항목

- 카카오디벨로퍼스 앱 생성
- 앱 이름 `다니엘의 노트`, 회사명 `플로렌스랩`, 실제 서비스 아이콘 등록
- 사업자 정보를 등록해 비즈 앱으로 전환
- 웹 플랫폼 도메인과 운영 콜백 URI 등록
- 카카오 로그인 활성화
- 프로필 닉네임과 카카오계정 이메일 동의항목 설정
- 필요하면 OpenID Connect 활성화
- 회원가입 화면, 개인정보처리방침, 탈퇴·연결 해제 화면을 심사용으로 준비

이메일 동의항목을 필수로 설정하려면 비즈 앱이 필요하다. 사용자가 카카오계정에 이메일을 등록하지 않았거나 제공 조건을 충족하지 않으면 이메일이 반환되지 않을 수도 있으므로, 카카오 고유 ID를 계정 식별의 기준으로 사용해야 한다.

## 필요한 운영 환경값

- `KAKAO_REST_API_KEY`
- `KAKAO_CLIENT_SECRET`
- `KAKAO_ADMIN_KEY`
- `KAKAO_REDIRECT_URI`

값이 모두 설정되고 계정 연결 마이그레이션이 끝난 뒤에만 로그인 화면에 노란색 `카카오로 계속하기` 버튼을 공개한다. 준비 전에는 작동하지 않는 버튼을 노출하지 않는다.

## 변경해야 할 공개 문서

- 개인정보처리방침: 카카오 로그인 수집 항목, 처리 목적, 위탁 또는 제3자 제공 관계
- 이용약관과 회원 안내: Google·카카오 로그인, 계정 연결, 탈퇴와 재가입 처리
- 마이페이지: 연결된 로그인 계정 확인 및 카카오 연결·해제

## 테스트 기준

- 카카오 신규 가입과 필수 동의
- 기존 카카오 회원 재로그인
- Google 회원의 카카오 계정 연결
- 동일 이메일 계정의 자동 병합 금지
- 구매 내역과 PDF 권한 유지
- 정지·탈퇴 회원 차단과 명시적 재가입
- 로그인 취소, 잘못된 `state`, 만료 코드, 토큰 검증 실패
- 카카오 연결 해제 후 기존 Google 로그인 유지
